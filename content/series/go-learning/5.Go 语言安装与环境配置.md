# 5. Go 语言安装与环境配置

## 引言

"工欲善其事，必先利其器"。在开始编写 Go 代码之前，我们需要先搭建好开发环境。

对于前端开发者来说，配置 Go 环境通常比配置 Node.js 环境要简单得多。你不需要安装 nvm 来管理版本（Go 的向后兼容性做得很好），也不需要配置复杂的 `.npmrc`（除了设置一个 Proxy）。

本篇文章将手把手教你如何在 macOS、Windows 和 Linux 上安装 Go，并配置 VS Code 作为开发工具。

## 1. 下载与安装

Go 语言的官方下载地址是 [go.dev/dl](https://go.dev/dl/)。

### 1.1 macOS 安装

你有两种选择：

**方式一：使用安装包（推荐新手）**
1. 下载 `.pkg` 文件（例如 `go1.25.0.darwin-arm64.pkg`，如果你是 M1/M2/M3 芯片）。
2. 双击运行，一路"下一步"即可。

**方式二：使用 Homebrew**
如果你习惯使用命令行工具：
```bash
brew install go
```

### 1.2 Windows 安装

1. 下载 `.msi` 安装包（例如 `go1.25.0.windows-amd64.msi`）。
2. 双击运行安装。
3. 安装程序会自动将 Go 的 bin 目录添加到你的系统环境变量 `PATH` 中。

### 1.3 Linux 安装

1. 下载 `.tar.gz` 包。
2. 解压到 `/usr/local`：
   ```bash
   rm -rf /usr/local/go && tar -C /usr/local -xzf go1.25.0.linux-amd64.tar.gz
   ```
3. 将 `/usr/local/go/bin` 添加到 `PATH` 环境变量中。

## 2. 环境变量配置

安装完成后，我们需要了解几个关键的环境变量。

### 2.1 关键变量解释

-   **GOROOT**: Go 语言的安装路径（例如 `/usr/local/go`）。通常不需要手动设置，Go 会自动识别。
-   **GOPATH**: Go 的工作区目录。在 Go Modules 出现之前，所有的代码都必须放在这里。现在虽然有了 Modules，但 `go install` 下载的二进制工具依然会放在 `$GOPATH/bin` 下。默认值为 `$HOME/go`。
-   **PATH**: 必须将 Go 的可执行文件路径加入系统 PATH，这样你才能在终端直接运行 `go` 命令。

### 2.2 配置 GOPROXY (重要！)

由于网络原因，国内用户直接访问 Go 的官方模块代理（proxy.golang.org）可能会非常慢甚至超时。**强烈建议**配置国内的镜像代理。

打开终端（Terminal 或 PowerShell），运行以下命令：

```bash
# 启用 Go Modules 功能
go env -w GO111MODULE=on

# 配置 GOPROXY 环境变量
go env -w GOPROXY=https://goproxy.cn,direct
```

> **注意**: `direct` 表示如果代理中找不到，则回源到源地址下载。

### 2.3 验证安装

打开一个新的终端窗口，输入：

```bash
go version
```

如果输出类似 `go version go1.25.0 darwin/arm64` 的信息，说明安装成功！

你还可以输入 `go env` 查看所有的环境配置。

## 3. 开发工具配置 (VS Code)

作为前端开发者，你大概率已经在使用 VS Code。好消息是，VS Code 对 Go 的支持非常完美。

### 3.1 安装 Go 插件

1. 打开 VS Code。
2. 点击左侧扩展图标（Extensions）。
3. 搜索 "Go"。
4. 安装由 **Go Team at Google** 开发的官方插件。

### 3.2 安装 Go 工具链

插件安装完成后，VS Code 可能会在右下角弹出一个提示："The 'gopls' command is not available..."。

1. 打开任意一个 `.go` 文件（如果没有，新建一个）。
2. 按 `Ctrl+Shift+P` (Mac: `Cmd+Shift+P`) 打开命令面板。
3. 输入 `Go: Install/Update Tools`。
4. 勾选所有列出的工具（如 `gopls`, `dlv`, `staticcheck` 等），点击确定。

这些工具包括：
-   **gopls**: Go Language Server，提供代码补全、跳转定义等核心功能。
-   **dlv**: 调试器。
-   **staticcheck**: 代码静态分析工具。

等待终端输出 "All tools successfully installed. You are ready to Go :)"。

## 4. Hello World：你的第一个 Go 程序

让我们按照惯例，写一个 Hello World。

### 4.1 创建项目目录

```bash
mkdir hello-go
cd hello-go
```

### 4.2 初始化 Go Module

在 Go 1.11 之后，我们使用 Go Modules 来管理项目。

```bash
go mod init example.com/hello
```
这会生成一个 `go.mod` 文件，类似于 Node.js 的 `package.json`。

### 4.3 编写代码

新建文件 `main.go`，输入以下内容：

```go
package main // 声明包名，main 包是程序的入口

import "fmt" // 导入 fmt 包，用于格式化输出

// main 函数是程序的入口点
func main() {
    fmt.Println("Hello, World! This is Go!")
}
```

### 4.4 运行程序

在终端中运行：

```bash
go run main.go
```

你应该能看到输出：`Hello, World! This is Go!`

### 4.5 编译程序

如果你想生成一个可执行文件：

```bash
go build
```

这会在当前目录下生成一个名为 `hello` (Windows 下是 `hello.exe`) 的二进制文件。

你可以直接运行它：
```bash
./hello
```

## 5. 理解 `go run` vs `go build`

-   **`go run`**: 类似于 `node index.js`。它会在临时目录编译并运行代码，运行完后删除二进制文件。适合开发调试。
-   **`go build`**: 类似于 `npm run build`（但更快）。它会生成最终的二进制文件，适合部署。

## 6. 总结

至此，你已经成功搭建了 Go 语言的开发环境，并运行了第一个程序。
